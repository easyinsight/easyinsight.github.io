Developing Against Easy Insight

Web Services

There's a lot more systems out there than Easy Insight can ever hope to directly connect to. To help accomodate passing in data from any custom system, Easy Insight offers a series of SOAP APIs to meet the differing requirements for publishing in custom data. You can use the API with any account type--the only difference is that as usual, the amount of space offered within your account, and the amount of API traffic allowed. Authentication to the web services is limited to basic auth for users at the free to group tiers. Full WS-Security authentication is offered to Professional and Enterprise accounts.

You can start by generating code against either of the WSDLs:

Unchecked API WSDL:

https://www.easy-insight.com/app/services/UncheckedPublishBasic?wsdl

Validated API WSDL:

https://www.easy-insight.com/app/services/ValidatedPublishBasic?wsdl

Getting Started

Conceptually, Easy Insight offers two primary APIs, the Unchecked and the Validated. The Unchecked API takes whatever data you send in and shapes a data source to match with that data. For example, if you pass in data for a data source called Business Data, and specify two key/value pairs, one with Revenue and a value of 500, the other with Region and a value of Northwest, the result of the call to the web service will be a new data source with a measure called Revenue and a grouping called Region, containing the appropriate values. You can use the Unchecked API to very quickly build up prototype applications and see what sort of integrations are possible from your data to Easy Insight.

For production code, it's recommended that you shift into using the Validated API. The Validated API provides identical methods to the Unchecked API, but with type safety and considerably better per-call performance. If you try to pass in a number to a field expecting a string, make a typo in a field name, or make any other similar error, the Validated API will reject the call, providing better reliability to your production functionality.
