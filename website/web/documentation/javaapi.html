What is the Easy Insight Java Wrapper?

The Easy Insight Java Wrapper is a convenience library we've thrown together to enable you to easily publish data into Easy Insight from Java code.

Where can I get the library?

Here! Download the Easy Insight Java Wrapper.

What external libraries do I need?

Absolutely nothing--this wrapper library uses nothing but core JDK classes.

Okay, how do I use the wrapper to publish data?

For this example, we're going to create a data source in Easy Insight and publish metrics out of the JVM into that data source.

First, we need to define the data source. The entry point to the API is the APIUtil.defineDataSource() call, which takes the data source name and your credentials. 


                    String myAPIKey = (Your Easy Insight username or API key);
                    String myAPISecretKey = (Your Easy Insight password or API secret key);
                    DataSourceFactory dataSourceFactory = APIUtil.defineDataSource("Sample JVM Data", myAPIKey,
                    myAPISecretKey);

Next, we need to add fields to the data source. There are a numble of calls available on DataSourceFactory to define different types of fields. We're going to add four fields to the data source:

  • Host -- our host name or IP address. This'll be a Grouping field.
  • Memory -- the memory used by the JVM. This'll be a Measure field.
  • Threads -- the number of threads running in the JVM. This'll be a Measure field.
  • Date -- the date/time at which we sent the data over.

                    dataSourceFactory.addGrouping("Host");
                    dataSourceFactory.addMeasure("Memory");
                    dataSourceFactory.addMeasure("Threads");
                    dataSourceFactory.addDate("Date");

Now that we've defined the fields, we need to actually make a call to Easy Insight to create or update the data source if it already exists in our account. We'll do that with: 


                    DataSourceOperationFactory dataSourceOperationFactory = dataSourceFactory.defineDataSource();

With the data source defined, we can actually add some data into the data source. There are several different operations available for adding new rows of data, replacing the entire data source contents with a new set of rows, or updating just a certain set of rows. For this example, we'll just add new rows to the data source: 


                    DataSourceTarget dataSourceTarget = dataSourceOperationFactory.addRowsOperation();
                    DataRow dataRow = dataSourceTarget.newRow();
                    dataRow.addValue("Host", InetAddress.getLocalHost().getHostAddress());
                    dataRow.addValue("Memory", Runtime.getRuntime().totalMemory());
                    dataRow.addValue("Threads", ManagementFactory.getThreadMXBean().getThreadCount());
                    dataRow.addValue("Date", new Date());
                    dataSourceTarget.flush();

At this point, the data source will be available to you under My Data in Easy Insight, with one row of data matching what we've sent.

How about composite data sources?

You can also define composite data sources inside of the wrapper. If you look at the examples directory of the zip, check out APICompositeExample for a sample of how to do this.

Is Javadoc for the library available?

Yep, check it out at http://www.easy-insight.com/documentation/javaapi.
