What's the Difference?

Easy Insight offers three APIs for loading data, and knowing the differences will help you decide which is best to use to load your data. The Unchecked API is the simplest, and the easiest to get started with - but it also has the potential to cause the most problems. The Unchecked API allows you to create a data set dynamically by name, if it doesn't exist, as well as rows. This may be ideal for some, but it can also lead to allowing malformed data into the system, as well as incorrect types.

The Validated API solves these issues by requiring you to set up a data source beforehand. The API is similar, but it will prevent you from creating erroneous fields, and notify you about the error.

The Dynamic API goes a step farther - a WSDL file is created specifically for the data source being connected to. The schema for the file is tied directly to the data source, automatically type-checking the rows added to the application at compile-time.

The Row Data Type

Unchecked API

addRow

Params:

  • dataSourceName - string - the name of the data source to be added to
  • row - row - the row to be added

The addRow method on the unchecked API adds a row to the data source named in the dataSourceName field. If there is no data source with that name, the data source will be created. If there are multiple data sources with that name, then the row is added to the first data source, so when using the unchecked API, use care in choosing data source names.
